04

없던 직군을,
워크플로우로.

PM·기획·UX라이터가 없어서, 직접 만들었다.

A solo designer with no PM, no PO, no UX writer — so I built the missing roles as a Claude Code workflow. AI drafts; the designer owns every decision.

문제 · 1인 조직엔 채울 직군이 없었다

PRD도, 문구 기준도,
아무도 안 만들었다.

위페어는 PM·PO·기획·UX라이터 없이 디자이너 혼자였다. 화면기능명세서만 주고받던 조직 — 결정의 근거도, 문구의 기준도 매번 처음부터 만들어야 했다. 사람으로 채울 수 없다면, 그 역할을 워크플로우로 만들 수 있을까?

No PM, no PO, no UX writer. Every decision and every label started from scratch. If the roles can’t be filled with people — can they be built as a workflow?

비어 있던 자리

없는 건 사람이 아니라,
반복 가능한 기준이었다.

PM
우선순위의 근거가 없다

영업이 들고 온 VOC가 메신저·구두로 흩어졌다. ‘무엇을 먼저’의 판단이 매번 1회성이라, 같은 논의가 반복됐다.

PO
PRD가 없다

요구사항이 화면기능명세 한 장으로만 오갔다. 기능 정의·플로우·예외가 문서로 남지 않아, 개발 핸드오프마다 구두 보완이 필요했다.

UXW
문구 기준이 없다

버튼·에러·안내 문구가 화면마다 제각각이었다. 톤과 규칙이 없으니 일관성은 매번 디자이너의 기억에 의존했다.

접근

AI가 초안, 사람이 판단.

빈 역할을 사람으로 채울 수 없으니, Claude Code 기반 Role-based 워크플로우로 구조화했다. 다만 자동화의 선을 분명히 그었다 — AI는 산출물의 초안까지, 모든 우선순위·정의·확정은 디자이너가 소유한다. 그래야 현장 맥락과 일관성이 흔들리지 않는다.

처음엔 UX 라이팅 봇만 만들려 했다. 만들다 보니 문제는 문구가 아니라 비어 있는 역할이었다.

역할 구성

네 개의 역할을, 하나의 흐름으로.

01
AI PM — 분석·우선순위

영업 VOC를 분류·정량화해 ‘무엇을 먼저’의 우선순위 초안을 만든다. 최종 우선순위는 현장 맥락을 아는 디자이너가 확정.

02
AI PO — 정의·플로우

선정된 기능의 정의·사용자 플로우·예외를 PRD 초안으로 구조화. 없던 PRD 프로세스를 0→1로 세웠다.

03
AI UX 라이터 — 문구

버튼·에러·안내 문구를 정해진 톤·규칙으로 일관되게 생성. UI 라이팅 기준을 0→1로 정립했다.

04
AI Reviewer — 일관성

산출물의 용어·톤·중복을 1차 점검해 사람의 검수 부하를 줄인다. 판단이 아니라 ‘체크’ 역할.

회고

자동화의 가치는, 판단에 집중하게 한 것.

LIMITATION · 정직
AI는 현장 맥락을 모른다
장갑 낀 손, 끊기는 네트워크 같은 비언어적 불편은 AI가 못 잡았다 — 현장 인터뷰로만 발견됐다. AI는 판단을 대신한 게 아니라 초안을 도왔을 뿐이다.
LEARNING
도구보다 방식이 먼저다
봇 하나가 아니라 ‘없는 역할’을 워크플로우로 정의한 게 핵심이었다. 도구를 늘리기 전에 일하는 방식을 먼저 설계했다.
